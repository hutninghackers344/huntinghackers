What is session hijacking?

Session hijacking is type of an cyber attack in which an attacker is able to steal users SESSION ID and it is also known as session stealing.

Now again the questions comes here that what is session ?

When a user logs into an web application then , it creates an session id which will created on the server , now session id are created because to track the state between multiple connections from the same user,instead of requesting the user to authenticate upon each click in a website.


Methods of Session Hijacking?

1.Session Sniffing:- Here an attacker will use wireshark tool or a proxy to capture the network traffic containing the session ID between the website and a cient .

2.Cross-Site Scripting:- Here an attacker will inject the malicious script in the website that is view by the users,now an attacker can make her script to show the session ID of all the users

3.Man-in-the-browser:- It is basically similar to man in the middle attack but the hacker needs to infects the victim system by trojan.Once the victim is tricked into installing the tojan onto the system, the trojan waits for the victim to visit a targeted site. The man-in-the-browser trojan can invisibly modify transaction information.

4.Session Sidejacking:- Here an attacker is able to find the vulnerablity in the victim browser and exploited it easily , now he can obtain an session ID by the browser.

5.Session fixation:- Here an attacker tricks her victim to use a specific session ID which basically provided by an attacker,. Once the user logs in, the attacker uses the known session ID to gain control of the user's session.
